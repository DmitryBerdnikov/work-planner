# Development Workflow

## 1. Создаем задачу

Описываем задачу в `docs/tasks/<short-task-name>.md` по шаблону из [tasks/README.md](tasks/README.md).

## 2. Планируем задачу

В новом чате включаем plan mode и указываем, с какой задачей работаем, например:

```text
Делаем задачу docs/tasks/example-task.md. Сначала исследуй и предложи план, без реализации.
```

План редактируем столько итераций, сколько нужно. В конце должен получиться итоговый план.

## 3. Реализуем задачу

После согласования плана запускаем реализацию. Во время реализации вносим правки по необходимости, но не расширяем scope без отдельного решения.

## 4. Делаем review

После реализации просматриваем `git diff`. Если нужны правки, вносим их в этом же чате и снова проверяем измененную часть.

## 5. Завершаем задачу

Когда задача принята:

1. Заполняем `Result` в task-файле.
2. Переносим task-файл в `docs/archive/`.
3. Обновляем `docs/roadmap.md`.

## 6. Commit и deploy

Вручную создаем commit. Название commit берем из имени
архивированной задачи, например:

```text
2026-06-11-review-status-command
```

Пушим в `main`, запускается автоматический deploy. Или делаем другие задачи и копим их для одного деплоя.
