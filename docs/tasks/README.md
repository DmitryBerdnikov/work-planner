# Tasks

Каждый файл в этом каталоге описывает одну активную самостоятельную задачу.

После выполнения задачи:

1. Заполнить `Result`.
2. Перенести файл в `docs/archive/`.
3. Обновить статус или ссылку в `docs/roadmap.md`.

## Template

```md
# Название задачи

## Problem

Что не так сейчас.

## Desired Behavior

Как должно работать.

## Current Context

Что уже известно по коду/докам.

## Plan

Конкретные шаги реализации.

## Tests

Какие тесты/проверки должны пройти.

## Risks

Что можно сломать.

## Result

Заполняется после выполнения. Здесь хранится короткая выжимка сделанной работы.
```
