# Analytics Reports

## Problem

Раздел отчетов запланирован, но analytics scope отложен до post-deploy.

## Desired Behavior

После production deploy появляется рабочий раздел отчетов: зарплата по периоду, количество рабочих записей и базовые агрегаты.

## Current Context

Бизнес-правила зарплаты описаны в `docs/rules-and-testing.md`: учитываются только прошедшие `work` записи, которые не отменены.

## Plan

- Уточнить минимальный набор отчетов для первого релиза.
- Реализовать shared pure helpers для агрегатов, если их не хватает.
- Добавить backend/query слой или frontend local aggregation по выбранному подходу.
- Добавить UI раздела отчетов.

## Tests

- `cancelled` запись не попадает в зарплату.
- `personal` запись не попадает в зарплату.
- Будущая `work` запись не попадает в зарплату.
- Прошедшая `work` запись попадает в выбранный период.

## Risks

- Можно задублировать бизнес-правила между frontend и backend.
- Нужно явно договориться о timezone для периодов.

## Result

Заполнить после реализации.
