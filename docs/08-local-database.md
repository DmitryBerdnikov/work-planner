# Локальная база данных

Work Planner использует один файл базы данных SQLite для локальной разработки:

```text
apps/api/.data/work-planner.sqlite
```

API создаёт директорию `.data` автоматически, но перед тем как приложение сможет обрабатывать реальные запросы, необходимо подготовить таблицы.

## Таблицы

Существует две группы таблиц:

- Таблицы Better Auth: `user`, `session`, `account`, `verification`.
- Таблицы приложения из Drizzle: `profiles`, `clients`, `appointments`, `attachments`.

## Команды

Из корня репозитория:

```bash
pnpm db:setup
```

Эта команда сначала запускает миграции Better Auth, а затем миграции Drizzle для таблиц приложения.

Для изменений схемы приложения:

```bash
pnpm db:generate
pnpm db:migrate
```

`pnpm db:generate` создаёт файлы миграций на основе `apps/api/src/db/schema.ts`.

`pnpm db:migrate` применяет миграции к базе данных SQLite, выбранной через `DATABASE_PATH`.

## Существующие базы данных, созданные вручную

Если локальная база данных была создана вручную до появления миграций, Drizzle может не знать, что её таблицы уже существуют. В таком случае используйте новую локальную базу данных для разработки или переместите старый файл перед запуском `pnpm db:setup`.

Не удаляйте вручную файлы баз данных staging или production.
