# Исправление веб-тестов

## Проблема

`pnpm --filter @work-planner/web test:quiet` завершался с ошибкой в тестах страницы календаря и выводил лишний шум от очистки Dexie в тестах локальной базы синхронизации.

## Ожидаемое поведение

Команда тихого запуска веб-тестов должна проходить успешно и не выводить лишние сообщения в stderr во время очистки тестовой базы данных.

## Текущий контекст

Тестовая фикстура календаря использует запись на май 2026 года. `CalendarPage` фильтрует записи по видимому диапазону FullCalendar. Тестовый мок не генерировал событие `datesSet`, поэтому страница сохраняла диапазон текущего месяца и отфильтровывала тестовую запись.

Тест локальной базы синхронизации удалял поддельные базы IndexedDB без предварительного закрытия активных подключений Dexie.

## План

- Настроить мок FullCalendar так, чтобы он генерировал видимый диапазон, включающий тестовую запись.
- Закрывать тестовые экземпляры Dexie перед удалением их баз данных.
- Проверить результат с помощью команды тихого запуска веб-тестов.

## Тесты

- `pnpm --filter @work-planner/web test:quiet`

## Риски

Низкие. Изменения ограничены тестами и скриптами пакетов.

## Результат

Добавлены скрипты `test:quiet` для пакетов рабочего пространства, исправлен мок FullCalendar в `CalendarPage`, чтобы он генерировал `datesSet` для недели с тестовой записью, и добавлено закрытие тестовых подключений Dexie перед удалением баз данных. Подтверждено, что `pnpm --filter @work-planner/web test:quiet` проходит успешно: выполнено 11 тестовых файлов и 49 тестов без лишних сообщений об очистке Dexie в stderr.
