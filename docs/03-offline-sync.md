# Offline-синхронизация

## Цель

Приложение должно позволять создавать и редактировать клиентов/записи без интернета. При восстановлении сети изменения отправляются на сервер.

## Подход

Frontend работает от локальной IndexedDB. Backend считается серверным источником истины после успешной синхронизации.

```text
React/Vite PWA
  -> IndexedDB/Dexie
  -> local changes/outbox
  -> /api/sync/push
  -> SQLite on VPS
  -> /api/sync/pull
```

## API

Минимальные sync endpoints:

```text
POST /api/sync/push
GET  /api/sync/pull
```

`push` отправляет локальные изменения на сервер.

`pull` получает изменения с сервера после последней успешной синхронизации.

## Правила

- ID создаются на клиенте через UUID, чтобы запись можно было создать offline.
- Offline-создание и редактирование поддерживается для клиентов и записей.
- Фото добавляются и изменяются только online.
- Изменения пишутся в локальный outbox.
- После успешного `push` локальные операции помечаются как синхронизированные.
- Удаления выполняются через `deleted_at`, не физическим удалением.
- Каждая синхронизируемая сущность имеет `updated_at` и `revision`.
- Конфликт решается по правилу last write wins.

## Важно для будущих полей

Клиент не должен без необходимости отправлять объект целиком. Для синхронизации предпочтительны операции/patch-изменения:

```text
entity
entity_id
operation
changed_fields
base_revision
client_timestamp
```

Это снижает риск, что старая версия приложения перезапишет новые поля.

## Фото

- Фото можно прикреплять только при наличии интернета.
- Если приложение offline, действие добавления фото недоступно.
- Backend принимает upload и возвращает `storage_key`.
- Лимиты: до 3 фото на запись, до 2 MB на фото после сжатия.
- Сжатие выполняется на frontend перед upload.
- Хранение на VPS выполняется локально через storage abstraction, чтобы позже перейти на S3-compatible storage.
